## Структура файлов
Насколько я понял, ты писал в процедурном стиле, буду иметь ввиду.
Название класса `Pictures`. Множественные имена в большинстве случаев плохо. Лучше назвать `Picture`. Но заглянув в сам файл класса, можно понять, что и название `Picture` не подходит. Ниже объясню почему.
Ридми довольно понятное, лаконичное, даже не нужно картинок. Респект!

## Main
`MAX_MISTAKES` и 'HIDDEN_LETTER_SYMBOL' не относятся к классу Main логически. Стоит положить в другой класс, там где эти константы уместны(Инкапсуляция в деле). И отсутствие полей тоже плохо, я бы туда положил какой-нибудь гейм стартер(например `Game`).
```java
public class Main {
// Неправильно
  private static final int MAX_MISTAKES = 7;
  private static final char HIDDEN_LETTER_SYMBOL = '□';
  
// Правильно
  private SomeGameClass game;
  private Menu menu;

}
```
## Pictures
Название класса не соответствует его функциональности. Он занимается еще отрисовкой человечка в зависимости от его стейта(или `stages`). Лучше назвать его HangmanRenderer/HangmanDrawer/HangmanPrinter.
